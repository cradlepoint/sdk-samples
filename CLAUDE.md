# Cradlepoint SDK Development — AI Instructions

## Project Overview

Ericsson NCOS SDK samples repository. Apps are Python 3.8 scripts that run on Cradlepoint routers (ARM64, musl libc) using `cppython`. The `cp` module provides access to router APIs.

## Repository Structure

```
apps/{category}/{app_name}/    # Sample apps organized by category
app_template/                   # Skeleton for new apps
web_app_template/               # Skeleton for web-based apps
archive/                        # Retired apps
docs/                           # API docs + app store site
make.py                         # Build/deploy tool
sdk_settings.ini                # Router connection settings
```

Categories: connectivity, monitoring, networking, integrations, gpio, vehicle, security, web_tools, examples

## Environment Setup

```bash
python3 make.py setup  # Creates .venv and installs dependencies
```

## Key Commands

```bash
.venv/bin/python make.py create {app_name}    # New app from template
.venv/bin/python make.py deploy {app_name}    # Full deploy to router
.venv/bin/python make.py build all            # Build all apps
.venv/bin/python make.py status {app_name}    # Check app on router
```

- `make.py` searches recursively under `apps/` — just pass the app name
- `deploy` handles purge + build + install + start automatically
- ALWAYS deploy after modifying app Python files

## Configuration

`sdk_settings.ini`:
```ini
[sdk]
app_name=your_app_name
dev_client_ip=192.168.0.1
dev_client_username=admin
dev_client_password=your_password
```

## Coding Rules (Python 3.8)

- Never use `str | None` — use `Optional[str]`
- Never use `print()` — use `cp.log()`
- Never use `input()` or `KeyboardInterrupt`
- Always use try/except, never bare `except:`
- Relative paths only (`tmp/`, never `/tmp`)
- Never modify packaged files (digital signatures)
- Never write default values to appdata
- `cppython` is the runtime, `start.sh` must use it

## CP Module Quick Reference

```python
import cp
cp.get('status/path')              # Read
cp.put('config/path', value)       # Update
cp.put('control/path', value)      # Action
cp.log('message')                  # Log
cp.get_appdata('field_name')       # Single appdata field
cp.put_appdata('name', 'value')    # Two separate args
cp.wait_for_wan_connection()       # Block until WAN up
```

- `cp.get()` returns data directly — NOT wrapped in `{"success": true, "data": ...}`
- Full API docs: `docs/ncos-api/`

## Web Apps

- Use Python's `http.server` module — never Flask/Bottle
- Default port 8000, always set SO_REUSEADDR
- Copy `static/` from `web_app_template/` for the design system
- Copy `your_web_app.html` as starting HTML

## Libraries

- Install to app folder: `.venv/bin/pip install -t path/to/app library`
- No .pyc or .so files — pure Python only
- `requests` is pre-installed on router — don't bundle it

## App Store

The app store site is at `docs/index.html` (deployed via GitHub Pages).
Catalog is generated by `docs/generate_catalog.py` from all `package.ini` files.
