This step-by-step wizard configures your NCM staging group for automatic device provisioning to NCX or NCS (SASE) networks. Use the navigation menu to progress through each configuration step, validate your settings, and apply the complete configuration to your staging group.

  • NCM v2 and v3 API key configuration
  • Real-time API connectivity validation
  • Staging and production group configuration
  • Exchange network, license, and DNS setup
  • Optional licenses and resource creation
  • Global Force DNS with per-device CSV override
  • Global site and resource tagging support
  • Automatic tag merging and deduplication
  • CSV/JSON bulk device configuration
  • Built-in file editor with CSV grid view
  • Comprehensive pre-deployment validation
  • One-click configuration deployment

    Step 1: API Keys

    Enter your NCM API credentials (v2 and v3) and test connectivity to ensure proper authentication with your NCM instance.

    Step 2: Required Parameters

    Configure essential settings including group IDs, exchange network ID, Secure Connect license, and DNS options.

    Step 3: Optional Parameters

    Select add-on licenses (SD-WAN, HMF, AI) and enable automatic resource creation for LAN subnets, FQDN hosts, and wildcard domains.

    Step 4: Global Tags

    Define global tags that will be automatically applied to all sites and resources created during the provisioning process.

    Step 5: Bulk Configuration

    Enable bulk device provisioning with CSV/JSON templates. Upload or edit files directly in the browser with grid view support.

    Step 6: Review & Apply

    Review your complete configuration summary, run final validation checks, and apply all settings to your staging group.
